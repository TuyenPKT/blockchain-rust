Quick Start Cài đặt

Yêu cầu: Rust 1.75+ và Cargo. Không cần dependencies ngoài.

1

Clone repository

$ git clone https://github.com/TuyenPKT/blockchain-rust.git
$ cd blockchain-rust
2

Cài Rust (nếu chưa có)

$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
$ source ~/.cargo/env
$ rustup update stable
3

Xem help & version timeline

$ cargo run
  PKT Blockchain — v4.8

  Cách dùng:
    cargo run -- wallet new             → tạo wallet mới
    cargo run -- wallet show            → xem địa chỉ ví
    cargo run -- mine [addr] [n]        → bắt đầu đào
    cargo run -- node <port> [peer]    → P2P node
    cargo run -- api [port]             → REST API (default 3000)
    cargo run -- explorer chain         → block explorer CLI
    cargo run -- genesis [regtest|testnet|mainnet]
    cargo run -- testnet [n] [port]     → local testnet
    cargo run -- metrics [node:port]    → hashrate, peers, mempool

  ── Era 1  (2009)  Bitcoin Genesis ──   v0.1 … v0.4
  ── Era 11 (2032) Optimization ──      v5.0 ✅ … v5.9 ✅
  ── Era 12 (2036) Multi-thread & GPU ── v6.0 … v6.9
  58+ versions. 12+ eras.
4

Tạo ví PKT & bắt đầu đào

# Tạo keypair mới, lưu ~/.pkt/wallet.key
$ cargo run -- wallet new
  Address      : 1PKTxxx...
  Pubkey hash  : a3f2c1... (dùng cho miner)
  Saved to     : ~/.pkt/wallet.key

# Mine tự động đến ví vừa tạo
$ cargo run -- mine

# Chạy 136 integration tests
$ cargo test
test result: ok. 57 passed; 0 failed
5

Chạy P2P node (tùy chọn)

Mở 2 terminal để test mạng ngang hàng:

# Terminal 1 — node lắng nghe port 8333
$ cargo run -- node 8333

# Terminal 2 — node kết nối tới node 1
$ cargo run -- node 8334 127.0.0.1:8333

Security Layer Stack Defense in Depth

Tầng Cơ chế Bảo vệ chống Mức độ
Classical Crypto ECDSA secp256k1, Schnorr, SHA-256 Classical computers 128-bit security
Post-Quantum (PQ) Dilithium FIPS-204, SPHINCS+ FIPS-205, ML-KEM FIPS-203 Quantum (Shor/Grover) 128-bit PQ security
Hybrid Transition ECDSA ∧ Dilithium (AND policy) Phải phá cả 2 cùng lúc Defense in depth
Zero-Knowledge Groth16, Recursive ZK/IVC, zkEVM Privacy leak khi verify Verifiable w/o witness
Privacy FHE-LWE, Pedersen Commitment, CoinJoin Plaintext trên chain Chain sees only ciphertext
Identity W3C DID + VC, DID Auth Central identity authority Self-sovereign
Availability Erasure coding (2x) + DAS sampling Data withholding attack 99.6% detection @ k=8

Khám phá từng module Source Code

Mỗi file trong src/ là một version độc lập — đọc tuần tự để theo dõi sự tiến hóa.

📦
block.rs + chain.rs
Block header, Merkle root, SHA-256 mining, difficulty adjustment mỗi 5 block. Genesis block hardcoded như Bitcoin thực.
💳
wallet.rs + hd_wallet.rs
ECDSA keypair, Base58Check address, BIP32 child key derivation, BIP39 mnemonic, BIP44 account/change/index path.
⚡
lightning.rs
Payment channel funding (2-of-2 P2SH), commitment TX off-chain, HTLC routing, cooperative/force close.
🌳
taproot.rs
Schnorr BIP340, tagged_hash, MAST (Merkelized Alternative Script Trees), P2TR output, MuSig2 key aggregation.
🔮
zk_proof.rs
Schnorr ZK interactive protocol, Hash Preimage proof, R1CS constraint system, Groth16 proving/verification.
🗜️
zk_rollup.rs + zkevm.rs
ZK-Rollup batch settlement trên L1, Optimistic Rollup fraud proof 7-day window, zkEVM per-opcode constraint generation.
⚛️
dilithium.rs + kyber.rs
CRYSTALS-Dilithium Module-LWE ring signature (NIST FIPS 204), ML-KEM key encapsulation với FO transform (NIST FIPS 203).
🌐
ibc.rs
ICS-02 light client, ICS-03 connection 4-way handshake, ICS-04 channel handshake, packet lifecycle, ICS-20 token transfer.
🆔
did.rs
W3C DID Core 1.0, did:chain:<hex> method, DID Document, VC issuance/verify, DID Auth challenge-response.
🔒
fhe_contract.rs
LWE-based Fully Homomorphic Encryption, HE-ADD/MUL-PLAIN, noise budget tracking, encrypted vote contract, salary sum, sealed bid auction.
🛸
sovereign_rollup.rs
DA layer với namespace blobs, Namespace Merkle Tree, erasure coding (2x), DAS sampling (99.6% @ k=8), sovereign upgrade.
🗺️
full_stack.rs
Version registry (58+v), era descriptions (12+ eras), stack statistics, security layers — tích hợp và demo tất cả từ v0.1 → v5.9.
🪙
wallet_cli.rs + miner.rs
PKT Wallet CLI: wallet new/show/address, lưu keypair tại ~/.pkt/wallet.key. Miner: live hashrate, difficulty auto-adjust, submit block qua P2P.
⚡
packetcrypt.rs
PacketCrypt PoW: announcement mining với nonce collision avoidance, block mining với effective_bits = base_bits − floor(log₂(ann_count+1)).
💾
storage.rs
RocksDB persistent storage: lưu/load chain + UTXO qua restart. Key schema: block:{height:016x}, utxo:{txid}:{index}, meta:height.
🌐
api.rs
REST API axum 0.7: GET /chain, GET /chain/:height, GET /balance/:addr, GET /mempool, POST /tx, GET /status, GET /metrics.
🔍
explorer.rs
Block Explorer CLI: explorer chain/block/tx/balance/utxo — đọc chain từ RocksDB, hiển thị 5 blocks gần nhất, tìm TX theo prefix.
🧪
genesis.rs
Testnet config: NetworkParams (magic bytes, coin params, ports), build_genesis(), local testnet N-node simulation. Bootstrap: seed.testnet.oceif.com:18333.
📊
metrics.rs
Runtime metrics: height, difficulty, UTXO count, mempool depth + fees, avg block time (20-block window), estimated hashrate (2^diff / block_time), peer count, sync status local vs remote. CLI: cargo run -- metrics · REST: GET /metrics.

Dependencies Cargo.toml

Minimal dependencies — chỉ cryptographic primitives cốt lõi, không có blockchain framework.

# Cargo.toml
[dependencies]
sha2       = "0.10"          # SHA-256, SHA-512
hex        = "0.4"           # hex encode/decode
chrono     = "0.4"           # timestamps
serde      = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
secp256k1  = { version = "0.27", features = ["rand-std", "global-context"] }
ripemd     = "0.1"           # RIPEMD-160 (Bitcoin address)
bs58       = "0.5"           # Base58Check encoding
tokio      = { version = "1", features = ["full"] }  # async P2P
hmac       = "0.12"          # HMAC-SHA512 (BIP32 HD wallet)
pbkdf2     = { version = "0.12", features = ["hmac"] }  # BIP39 mnemonic
rocksdb    = "0.21"          # v4.2: persistent chain + UTXO storage
axum       = "0.7"           # v4.4: REST API (GET /chain, POST /tx, ...)
